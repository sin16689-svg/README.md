# 리눅스 명령어

## 1. top
- 시스템의 실시간 성능 상태와 프로세스 목록을 보여주는 명령어

---

### System Summary(상단 부분):
시스템에 대한 전체적인 상태 요약 정보

- 현재 시간, 시스템 부팅 시간 및 업타임
- CPU, 메모리, 스왑 사용량(사용량, 가용량, 전체 용량)
- 프로세스 총 수, 실행 중인 프로세스 수, 슬립 중인 프로세스수
- CPU 사용량(user, system, idel)
  
### Task/Process List(하단 부분):

- 각 프로세스에 대한 상세 정보(PID, 사용자, CPU사용량, 메모리 사용량 등)

---

### 주요 키 설명

| 주요 키 | 기능 설명 |
|----|-----------|
| **q** | top 종료 |
| **h** | 도움말 표시(사용 가능한 명령어 목록 출력) |
| **P** | CPU 사용률 기준 정렬 |
| **M** | 메모리 사용량 기준 정렬 |
| **T** | 실행 시간 기준 정렬 |
| **k** | 프로세스 종료(입력한 PID를 종료) |
| **r** | 프로세스 우선순위 변경(PID + 새로운 우선순위 입력) |
| **z** | 컬러 모드 전환 |
| **c** | 명령어 라인 표시/숨기기(프로세스 실행 커맨드 표시) |
| **1** | CPU 코어별 사용량 표시/숨기기 |
| **u** | 특정 사용자 프로세스만 보기 |
| **i** | idle 프로세스 숨기기 |
| **d / s** | 갱신 주기(초) 변경 |
| **Shift + R** | 정렬 순서 뒤집기 |
| **f** | 컬럼 선택 메뉴 |
| **o** | 정렬 기준 변경 |

---

### 명령어 옵션 

| 옵션 | 설명 |
|------|--------------------------|
| `-d 초` | 새로고침 주기 변경 |
| `-u 사용자` | 특정 사용자 프로세스만 표시 |
| `-p PID` | 특정 프로세스만 모니터링 |
| `-n 횟수` | 지정된 횟수만 갱신 후 종료 |

---

### 활용 예시 

```bash
top               # 기본 실행
top -d 1          # 1초마다 갱신
top -u student    # 특정 사용자 프로세스 보기
top -p 1234       # 특정 PID 모니터링
```

### 활용 상황

- 서버 CPU 사용률이 갑자기 증가했을 때 원인 프로세스 확인
- 메모리 누수 의심 프로세스 추적
- 특정 프로그램의 리소스 사용 추세 모니터링
- 백그라운드에서 리소스 과사용 프로세스 탐지

## 2. ps
- 'Process Status' 약자, 현재 실행중인 프로세스들의 상태를 보여주는 명령어

---

### 역사

- 유닉스 초기부터 존재한 매우 오래된 명령어
- **Ken Thompson(유닉스 창시자)**가 초기 시스템 관리 도구의 하나로 만들었음
- 초창기에는 프로세스 정보를 단순하게 보여주는 용도로 설계됨
- 이후 BSD, System V를 거치며 옵션 체계가 확장됨 
- 현재 Linux의 ps는 BSD·System V·GNU 옵션이 섞인 형태로 발전함
  
---

### 주요 옵션

| 주요 옵션 |   설명   |
|----|-----------|
| **a** | 터미널에 종속되지 않은 모든 사용자 프로세스 표시 |
| **u** | 실행 사용자 정보 포함 표시 |
| **x** | 터미널 연결 없는 프로세스도 표시 |
| **e** | 모든 프로세스 표시 (-A와 동일) |
| **f** | 프로세스를 트리(Tree) 형태로 표시 |
| **o** | 프로세스 종료(입력한 PID를 종료) |
| **p[PID]** | 원하는 출력 컬럼만 선택 |
| **--forest** | 트리 구조로 계층 표시 (BSD/Linux) |
| **eo[columns]** | 출력 컬럼을 자유롭게 지정 가능 (예: ps -eo pid,user,cmd) |
| **ef** | System V 스타일의 전체 프로세스 상세 출력 |

---

### ps aux 컬럼 

| 컬럼 | 설명 |
|----|-----------|
| **USER** | 프로세스를 실행한 사용자 |
| **PID** | 프로세스 ID |
| **%CPU** | CPU 사용률 |
| **%MEM** | 메모리 사용률 |
| **VSZ** | 가상 메모리 사용량 |
| **RSS** | 실제 메모리 사용량 |
| **TTY** | 실행 터미널 |
| **STAT** | 프로세스 상태 코드 |
| **START** | 프로세스 시작 시간 |
| **TIME** | 누적 CPU 사용 시간 |
| **COMMAND** | 실행된 프로그램/명령어 전체 경로로 |

---

### 활용 예시 

```bash
ps aux                # 모든 프로세스 보기
ps aux | grep httpd   # httpd 관련 프로세스 찾기
ps -ef --forest       # 트리 형태로 보기
ps -eo pid,cmd        # PID + 명령어만 보기
ps -p 1234            # PID 1234만 출력
```

### 활용 상황

- 특정 프로세스가 실행 중인지 확인할 때
- 비정상적으로 종료되지 않는 프로세스 조사
- 좀비 프로세스(Z 상태) 확인
- kill 명령어로 종료할 PID 정보 확인

## 3. jobs
- 현재 실행 중인 백그라운드 작업(프로세스)을 조회하는 명령어
- '&'로 실행한 백그라운드 작업, 'ctrl+Z'로 중단된 작업 등을 확인할 때 사용
- 주로 **프로세스를 다시 포그라운드(fg)로 가져오거나 종료(kill)할 때 참고
  
 ---

 ### 주요 옵션 

| 주요 옵션 |   설명   |
|----|-----------|
| **-l** | PID를 포함하여 상세 정보 출력 |
| **-p** | PID만 출력 |
| **-r** | 실행중인(Running) 작업만 표시 |
| **-s** | 중단된(Stopped) 작업만 표시 |

---

### 활용 예시

```bash
sleep 100 &      # 백그라운드 작업 실행
jobs             # 백그라운드 작업 조회
jobs -l          # PID 포함 상세 조회
fg %1            # 1번 작업을 포그라운드로 가져오기
bg %1            # 1번 작업을 백그라운드로 계속 실행
```

---

### 활용 상황

- 터미널에서 여러 작업을 동시에 실행할 때 관리용
- 중단된 작업을 다시 실행하거나 종료할 때
- 포그라운드와 백그라운드 전환 확인
* 현재 쉘 세션에서만 작동함 - 다른 터미널에서 실행한 백그라운드 프로세스는 표시 되지 않음
  
---

## 4. kill
- 프로세스에 시그널을 보내는 명령어
- 프로세스를 종료하거나 특정 시그널을 전달할 때 사용
- 일반적으로 **종료 시그널(SIGTERM, 15)**을 보내는 것이 기본
- 강제 종료 **SIGKILL(9)을 사용

---

### 주요 옵션

| 주요 옵션 |   설명   |
|----|-----------|
| **-l** | 사용 가능한 시그널 목록 표시 |
| **-s <SIGNAL>** | 특정 시그널을 지정하여 전송(예: SIGKILL, SIGTERM) |
| **-SIGNAL** | 시그널을 바로 지정 가능(예: kill -9 PID) |
| **-q <SIGNAL>** | 시그널 전송 후 메세지 출력 여부 조절 |

---

### 주요 시그널 

| 시그널 |   번호   | 설명|
|----|-------|-------------|
| **SIGTERM** | 15 | 기본 종료 시그널, 프로세스가 정리할 시간 제공 |
| **SIGKILL** | 9 | 강제 종료, 프로세스가 무조건 종료 됨 |
| **SIGSTOP** | 19 | 프로세스 정지 |
| **SIGCONT** | 18 | 정지된 프로세스 재개 |

---

### 활용 예시

```bash
kill 1234          # PID 1234 종료 (SIGTERM 기본)
kill -9 1234       # PID 1234 강제 종료 (SIGKILL)
kill -s SIGSTOP 1234 # PID 1234 일시정지
kill -s SIGCONT 1234 # PID 1234 재개
kill -l            # 시그널 목록 확인
```

---

### 활용 상황

- 비정상적으로 멈춘 프로세스 종료
- 특정 백그라운드 작업 강제 종료
- 프로세스 상태 제어(정지, 재개)
- PID 기반 프로세스 관리
